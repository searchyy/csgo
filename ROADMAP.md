# CS2 炼金扫描器产品路线

## 目标

项目最终提供 3 个核心界面：

1. **多平台价格看板**
   - 展示当前 CS2 饰品在多个平台的价格
   - 支持中文名称搜索、平台筛选、外观筛选、StatTrak 筛选
   - 数据来源优先读取本地 `SQLite` 快照缓存

2. **全量炼金 EV 排行**
   - 基于真实静态饰品数据库、真实价格缓存、真实磨损公式
   - 扫描可参与炼金的目标饰品
   - 输出每个目标的候选配方、成本、成功率、EV、ROI
   - 支持按套利率从高到低排序

3. **单目标最优炼金方案**
   - 输入目标枪械和目标外观
   - 自动逆向生成配方
   - 给出最优材料组合、目标概率、卡磨损要求、收益表现

## 当前已完成

### 1. 静态数据库

- 已接入真实静态饰品库
- 当前 `data/items.json` / `data/items.sqlite` 已包含真实：
  - `collection`
  - `rarity`
  - `min_float`
  - `max_float`
  - `available_variants`
  - `available_exteriors`

### 2. 价格抓取与缓存

- 已接入 `SteamDT` 私有市场抓取
- 已支持抓取结果写入 `data/steamdt_prices.sqlite`
- 已支持缓存优先、缺失回源、写回本地库
- 已支持纯缓存模式，避免每次都联网

### 3. 炼金计算核心

- 已实现：
  - 磨损计算
  - 概率分布
  - EV / ROI 计算
  - 逆向配方生成
  - 单目标扫描

## 统一数据流

### A. 数据层

1. 静态库同步
   - 来源：ByMykel CSGO API
   - 落地：`data/items.sqlite`

2. 市场价格抓取
   - 来源：SteamDT / 未来可扩展 BUFF、悠悠有品、HaloSkins
   - 落地：`data/steamdt_prices.sqlite`

3. 扫描结果缓存
   - 下一步建议新增：
     - `data/scan_results.sqlite`
     - 用于缓存全量 EV 扫描结果

### B. 计算层

1. 价格看板服务
   - 从价格快照库聚合每把枪每个外观每个平台价格

2. EV 扫描服务
   - 从静态库读取真实升级链
   - 从价格缓存读取材料价和产出价
   - 计算候选配方 EV / ROI

3. 单目标优化服务
   - 输入目标饰品
   - 输出最优配方列表和详细拆解

### C. 展示层

建议统一做一个本地 Web 应用，包含 3 个页面：

1. `/dashboard/prices`
2. `/dashboard/ev`
3. `/optimizer`

## 建议实施顺序

### 第一阶段：先把数据抓全

目标：

- 把更多枪械价格持续抓到 `SQLite`
- 支持定时增量同步
- 控制抓取速率，避免风控

建议补充：

- 全量抓取任务调度器
- 随机休眠 / 分批抓取 / 失败重试
- 抓取日志和进度统计

### 第二阶段：做统一 Web 面板

目标：

- 用一个前端统一展示现有 `SQLite` 数据
- 替代当前静态 HTML 导出方式

建议页面：

- 价格看板页
- EV 排行页
- 单枪优化页

### 第三阶段：做全量 EV 扫描缓存

目标：

- 不只是临时扫单个目标
- 支持批量扫描全量热门目标
- 扫描结果落库后前端直接读取

建议新增表：

- `scan_runs`
- `scan_results`
- `scan_result_materials`
- `scan_result_outcomes`

## 现在最合理的产品形态

### 页面 1：多平台价格看板

输入：

- 搜索词
- 外观
- 是否 StatTrak
- 平台

输出：

- 中文枪名
- 收藏品系列
- 稀有度
- 各平台价格
- 最近抓取时间

### 页面 2：全量炼金 EV 排行

输入：

- ROI 下限
- 目标稀有度
- 最大配方数量
- 仅看热门目标 / 全量目标

输出：

- 目标枪械
- 目标外观
- 配方
- 成本
- EV
- ROI
- 目标概率
- 材料列表
- 主要产出列表

### 页面 3：单目标最优方案

输入：

- 目标枪械
- 目标外观

输出：

- 最优方案列表
- 目标收藏品路线
- 材料磨损上限
- 建议买入外观
- 产出分布
- EV / ROI
